## Installation

<Steps>
  <Step title="Clone the Repository">
    ```bash
    git clone https://github.com/your-org/bookline.git
    cd bookline
    ```
  </Step>

  <Step title="Install Dependencies">
    ```bash
    npm install
    ```
    
    This installs all required packages:
    - `express` - Web server
    - `@supabase/supabase-js` - Database client
    - `razorpay` - Payment gateway SDK
    - `axios` - HTTP client for WhatsApp API
    - `node-cron` - Job scheduling
    - `fuse.js` - Fuzzy search
  </Step>

  <Step title="Configure Environment Variables">
    Copy the example environment file:
    
    ```bash
    cp .env.example .env
    ```
    
    Edit `.env` with your credentials:
    
    ```bash .env
    # Server Configuration
    PORT=3000
    NODE_ENV=development

    # Supabase
    SUPABASE_URL=https://your-project.supabase.co
    SUPABASE_SERVICE_ROLE_KEY=your-service-role-key

    # Meta WhatsApp Cloud API
    WHATSAPP_VERIFY_TOKEN=your-webhook-verify-token
    WHATSAPP_API_VERSION=v21.0

    # Razorpay
    RAZORPAY_KEY_ID=your-razorpay-key-id
    RAZORPAY_KEY_SECRET=your-razorpay-key-secret
    RAZORPAY_WEBHOOK_SECRET=your-razorpay-webhook-secret

    # Order Expiry (minutes)
    ORDER_EXPIRY_MINUTES=10
    ```
    
    <Warning>
      The `WHATSAPP_VERIFY_TOKEN` can be any random string you choose. You'll use this same value when configuring the Meta webhook.
    </Warning>
  </Step>

  <Step title="Set Up Database">
    Go to your Supabase project dashboard:
    
    1. Navigate to **SQL Editor**
    2. Create a new query
    3. Copy the contents of `supabase/migration.sql`
    4. Run the query
    
    This creates all necessary tables and functions:
    
    <Accordion title="Database Schema Overview">
      - `clinics` - Clinic information and WhatsApp credentials
      - `doctors` - Doctor profiles linked to clinics
      - `doctor_configs` - Fee, capacity, and booking window settings
      - `doctor_daily_states` - Daily runtime state (queue counter)
      - `orders` - Payment lifecycle tracking
      - `bookings` - Confirmed bookings with position
      - `admin_sessions` - Admin authentication sessions
      - `conversation_states` - Multi-step WhatsApp conversation tracking
    </Accordion>
  </Step>

  <Step title="Deploy to a Public URL">
    BookLine needs a public HTTPS URL for webhooks. Choose one:
    
    <Tabs>
      <Tab title="Render (Recommended)">
        1. Push your code to GitHub
        2. Go to [render.com](https://render.com)
        3. Create a new **Web Service**
        4. Connect your repository
        5. Set environment variables
        6. Deploy
        
        Render provides a free HTTPS URL like `https://bookline-xyz.onrender.com`
      </Tab>
      
      <Tab title="ngrok (Development)">
        For local testing:
        
        ```bash
        # In one terminal, start BookLine
        npm run dev
        
        # In another terminal, expose it
        ngrok http 3000
        ```
        
        Use the ngrok HTTPS URL for webhook configuration.
        
        <Warning>
          ngrok URLs change on each restart. Not suitable for production.
        </Warning>
      </Tab>
    </Tabs>
  </Step>

  <Step title="Configure WhatsApp Webhook">
    In the Meta Developer Console:
    
    1. Go to your WhatsApp Business app
    2. Navigate to **WhatsApp > Configuration**
    3. Click **Edit** next to Webhook
    4. Enter your webhook details:
       - **Callback URL**: `https://your-domain.com/webhook`
       - **Verify Token**: Same value as `WHATSAPP_VERIFY_TOKEN` in `.env`
    5. Click **Verify and Save**
    6. Subscribe to `messages` field
    
    <Note>
      The verify token is used to authenticate Meta's webhook verification request. It can be any random string.
    </Note>
  </Step>

  <Step title="Configure Razorpay Webhook">
    In the Razorpay Dashboard:
    
    1. Go to **Settings > Webhooks**
    2. Click **Create New Webhook**
    3. Enter webhook details:
       - **URL**: `https://your-domain.com/payment/webhook`
       - **Secret**: Same value as `RAZORPAY_WEBHOOK_SECRET` in `.env`
       - **Active Events**:
         - `payment.captured`
         - `order.paid`
    4. Save
    
    <Warning>
      Keep the webhook secret secure. BookLine uses it to verify that payment notifications are genuinely from Razorpay.
    </Warning>
  </Step>

  <Step title="Start the Server">
    ```bash
    npm start
    ```
    
    You should see:
    
    ```
    🏥 BookLine Server running on port 3000
    📡 Environment: production
    🔗 https://your-domain.com/health
    📱 https://your-domain.com/webhook
    💳 https://your-domain.com/payment/webhook
    ```
    
    For development with auto-restart:
    
    ```bash
    npm run dev
    ```
  </Step>
</Steps>

## Verify Installation

<Steps>
  <Step title="Health Check">
    Visit `https://your-domain.com/health` in your browser. You should see:
    
    ```json
    {
      "status": "ok",
      "timestamp": "2026-03-04T12:00:00.000Z"
    }
    ```
  </Step>

  <Step title="Send Test Message">
    Send "hi" to your WhatsApp Business number. The bot should respond with a greeting or search prompt.
    
    <Note>
      If you haven't registered any clinics/doctors yet, you'll need to do that first via the registration flow.
    </Note>
  </Step>
</Steps>

## Common Issues

<AccordionGroup>
  <Accordion title="Webhook verification fails">
    **Problem**: Meta shows "Verification Failed" when adding webhook.
    
    **Solutions**:
    - Ensure your server is publicly accessible via HTTPS
    - Verify that `WHATSAPP_VERIFY_TOKEN` in `.env` matches the token you entered in Meta console
    - Check server logs for incoming verification requests
    - Test with: `curl "https://your-domain.com/webhook?hub.mode=subscribe&hub.verify_token=your-token&hub.challenge=test"`
  </Accordion>
  
  <Accordion title="Bot doesn't respond to messages">
    **Problem**: Messages sent to WhatsApp number receive no response.
    
    **Solutions**:
    - Check that webhook is subscribed to `messages` field in Meta console
    - Verify server logs show incoming POST requests to `/webhook`
    - Ensure no firewall is blocking Meta's IP ranges
    - Check Supabase connection is working
  </Accordion>
  
  <Accordion title="Payment webhook not processing">
    **Problem**: Payments succeed but bookings aren't created.
    
    **Solutions**:
    - Verify webhook URL in Razorpay dashboard is correct
    - Check that `RAZORPAY_WEBHOOK_SECRET` matches
    - Review server logs for webhook signature validation errors
    - Ensure orders table has the corresponding `gateway_order_id`
  </Accordion>
  
  <Accordion title="Database connection errors">
    **Problem**: `SUPABASE_URL` or `SUPABASE_SERVICE_ROLE_KEY` errors.
    
    **Solutions**:
    - Verify credentials in Supabase dashboard under **Settings > API**
    - Use the **service_role** key, not the **anon** key
    - Ensure your Supabase project is active (not paused)
    - Check for typos in `.env` file
  </Accordion>
</AccordionGroup>

## Development Mode

For local development with auto-restart:

```bash
npm run dev
```

This uses Node's `--watch` flag to automatically restart the server when files change.

<Tip>
  Use ngrok or a similar tunnel service to expose your local server for webhook testing during development.
</Tip>

## Environment Variables Reference

| Variable | Required | Description | Example |
|----------|----------|-------------|----------|
| `PORT` | No | Server port | `3000` |
| `NODE_ENV` | No | Environment mode | `development` or `production` |
| `SUPABASE_URL` | Yes | Supabase project URL | `https://abc123.supabase.co` |
| `SUPABASE_SERVICE_ROLE_KEY` | Yes | Service role key from Supabase | `eyJhbG...` |
| `WHATSAPP_VERIFY_TOKEN` | Yes | Custom token for webhook verification | Any random string |
| `WHATSAPP_API_VERSION` | No | Meta API version | `v21.0` (default) |
| `RAZORPAY_KEY_ID` | Yes | Razorpay key ID | `rzp_test_...` |
| `RAZORPAY_KEY_SECRET` | Yes | Razorpay secret key | `abc123...` |
| `RAZORPAY_WEBHOOK_SECRET` | Yes | Razorpay webhook secret | `xyz789...` |
| `ORDER_EXPIRY_MINUTES` | No | Order expiry time | `10` (default) |

<Warning>
  Never commit your `.env` file to version control. Add it to `.gitignore`.
</Warning>
