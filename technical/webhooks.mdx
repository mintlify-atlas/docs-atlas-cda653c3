BookLine uses webhooks to receive real-time notifications from WhatsApp Cloud API and Razorpay. This guide covers webhook setup, verification, and testing.

## Overview

BookLine implements two webhook endpoints:

- **WhatsApp Webhook**: `/webhook` - Receives incoming messages and status updates
- **Razorpay Webhook**: `/payment/webhook` - Receives payment confirmation events

## WhatsApp Webhook

### Endpoint Configuration

<Steps>
  <Step title="Set up webhook URL">
    Configure your webhook URL in Meta Developer Console:
    
    ```bash
    https://yourdomain.com/webhook
    ```
    
    The endpoint must be publicly accessible via HTTPS.
  </Step>

  <Step title="Configure verify token">
    Set the `WHATSAPP_VERIFY_TOKEN` in your environment:
    
    ```bash
    WHATSAPP_VERIFY_TOKEN=your-secure-random-token
    ```
    
    This token is used to verify webhook requests from Meta.
  </Step>

  <Step title="Subscribe to webhook events">
    In Meta Developer Console, subscribe to:
    - `messages` - Incoming text and interactive messages
  </Step>
</Steps>

### Verification Logic

When Meta verifies your webhook, it sends a GET request with query parameters:

```javascript src/routes/webhook.js:13-25
router.get('/', (req, res) => {
    const mode = req.query['hub.mode'];
    const token = req.query['hub.verify_token'];
    const challenge = req.query['hub.challenge'];

    if (mode === 'subscribe' && token === env.WHATSAPP_VERIFY_TOKEN) {
        console.log('[Webhook] ✅ Verification successful');
        return res.status(200).send(challenge);
    }

    console.warn('[Webhook] ❌ Verification failed');
    return res.sendStatus(403);
});
```

<ParamField query="hub.mode" type="string" required>
  Must be `subscribe` for webhook verification
</ParamField>

<ParamField query="hub.verify_token" type="string" required>
  Must match your `WHATSAPP_VERIFY_TOKEN` environment variable
</ParamField>

<ParamField query="hub.challenge" type="string" required>
  Random string that must be echoed back to complete verification
</ParamField>

### Message Processing

BookLine processes both text and interactive messages:

```javascript src/routes/webhook.js:67-90
for (const msg of messages) {
    const from = msg.from; // sender phone number
    const messageId = msg.id;
    const messageType = msg.type;

    let messageBody = '';
    let interactiveData = null;

    if (messageType === 'text') {
        messageBody = msg.text?.body || '';
    } else if (messageType === 'interactive') {
        interactiveData = msg.interactive;
    } else {
        // Ignore other message types (images, audio, etc.)
        continue;
    }

    // Route to handler (fire and forget — we already sent 200)
    messageRouter
        .route(phoneNumberId, from, messageId, messageBody, messageType, interactiveData)
        .catch(err => {
            console.error('[Webhook] Message handler error:', err);
        });
}
```

<Note>
  BookLine responds with `200 OK` immediately to prevent Meta retries, then processes messages asynchronously.
</Note>

## Razorpay Webhook

### Endpoint Configuration

<Steps>
  <Step title="Configure webhook URL in Razorpay Dashboard">
    Go to Settings → Webhooks and add:
    
    ```bash
    https://yourdomain.com/payment/webhook
    ```
  </Step>

  <Step title="Set webhook secret">
    Copy the webhook secret from Razorpay and add to your environment:
    
    ```bash
    RAZORPAY_WEBHOOK_SECRET=your-webhook-secret
    ```
  </Step>

  <Step title="Subscribe to events">
    Enable the following events:
    - `payment.captured` - Standard payment success
    - `order.paid` - Order payment confirmation
    - `payment_link.paid` - Payment link completion
    - `qr_code.credited` - UPI QR code payment
  </Step>
</Steps>

### Signature Verification

Razorpay signs webhook requests using HMAC SHA256:

```javascript src/routes/payment.js:23-37
const signature = req.headers['x-razorpay-signature'];
const rawBody = Buffer.isBuffer(req.body) 
    ? req.body.toString('utf8') 
    : (typeof req.body === 'string' ? req.body : JSON.stringify(req.body));

const expectedSignature = crypto
    .createHmac('sha256', env.RAZORPAY_WEBHOOK_SECRET)
    .update(rawBody)
    .digest('hex');

if (signature !== expectedSignature) {
    console.warn('[Payment] ❌ Invalid webhook signature');
    return res.status(400).json({ error: 'Invalid signature' });
}
```

<Warning>
  Always verify webhook signatures to prevent fraudulent payment confirmations.
</Warning>

### Payment Event Handling

BookLine handles multiple payment event types:

```javascript src/routes/payment.js:43-68
const eventType = event.event;

// We only care about successful payments
if (eventType !== 'payment.captured' && eventType !== 'order.paid'
    && eventType !== 'payment_link.paid' && eventType !== 'qr_code.credited') {
    return res.status(200).json({ status: 'ignored' });
}

// Extract order info
let gatewayOrderId;
let paymentId;
let orderId; // our internal order ID (from QR notes)

if (eventType === 'payment_link.paid') {
    gatewayOrderId = event.payload?.payment_link?.entity?.id;
    paymentId = event.payload?.payment?.entity?.id;
} else if (eventType === 'qr_code.credited') {
    // QR payments: find our order via notes
    paymentId = event.payload?.payment?.entity?.id;
    orderId = event.payload?.qr_code?.entity?.notes?.order_id;
} else if (eventType === 'payment.captured') {
    gatewayOrderId = event.payload?.payment?.entity?.notes?.payment_link_id
        || event.payload?.payment?.entity?.order_id;
    paymentId = event.payload?.payment?.entity?.id;
}
```

### Idempotency Protection

BookLine prevents duplicate processing using optimistic concurrency:

```javascript src/routes/payment.js:89-101
// Idempotency check — only process if status is 'pending'
if (order.status !== 'pending') {
    console.log(`[Payment] Order already processed (status: ${order.status})`);
    return res.status(200).json({ status: 'already_processed' });
}

// Mark order as paid (with optimistic concurrency)
const updatedOrder = await orderService.updateStatus(order.id, 'pending', 'paid');
if (!updatedOrder) {
    // Another webhook already processed this
    console.log(`[Payment] Concurrent processing — skipping`);
    return res.status(200).json({ status: 'concurrent_skip' });
}
```

<Note>
  Razorpay may send duplicate webhooks. BookLine uses database-level concurrency control to ensure each payment is processed only once.
</Note>

### Common Issues

<AccordionGroup>
  <Accordion title="WhatsApp verification fails">
    **Symptoms**: Meta shows "Verification failed" error
    
    **Solutions**:
    - Ensure `WHATSAPP_VERIFY_TOKEN` matches exactly
    - Check webhook URL is publicly accessible via HTTPS
    - Verify no authentication middleware blocks GET requests
  </Accordion>

  <Accordion title="Razorpay signature mismatch">
    **Symptoms**: `Invalid webhook signature` error in logs
    
    **Solutions**:
    - Verify `RAZORPAY_WEBHOOK_SECRET` is correct
    - Ensure raw body parser is used: `express.raw({type: 'application/json'})`
    - Check webhook secret hasn't been rotated in Razorpay dashboard
  </Accordion>

  <Accordion title="Messages not processing">
    **Symptoms**: Webhooks received but no response sent
    
    **Solutions**:
    - Check `phone_number_id` matches clinic configuration
    - Verify WhatsApp access token is valid
    - Check messageRouter logs for handler errors
  </Accordion>

  <Accordion title="Duplicate payment processing">
    **Symptoms**: Same payment processed multiple times
    
    **Solutions**:
    - Verify order status checks are working
    - Check database constraints on order status updates
    - Review optimistic concurrency implementation
  </Accordion>
</AccordionGroup>
