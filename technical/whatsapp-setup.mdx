BookLine uses Meta's WhatsApp Cloud API to send and receive messages. This guide covers account setup, phone number configuration, and access token management.

## Step 1: Create Meta App

<Steps>
  <Step title="Go to Meta Developer Console">
    Visit [developers.facebook.com](https://developers.facebook.com) and sign in with your Facebook account.
  </Step>

  <Step title="Create new app">
    - Click **Create App**
    - Select **Business** as app type
    - Fill in app name and contact email
    - Link to your Meta Business Account
  </Step>

  <Step title="Add WhatsApp product">
    - In your app dashboard, click **Add Product**
    - Find **WhatsApp** and click **Set up**
    - Select your Business Account
  </Step>
</Steps>

## Step 2: Configure Phone Number

<Steps>
  <Step title="Add phone number">
    In WhatsApp → Getting Started:
    - Click **Add phone number**
    - Choose between:
      - **Test number** (for development, limited to 5 recipients)
      - **Production number** (requires business verification)
  </Step>

  <Step title="Get Phone Number ID">
    Copy your **Phone Number ID** from the dashboard:
    
    ```bash
    123456789012345
    ```
    
    This ID is used in WhatsApp API calls, stored per-clinic in BookLine.
  </Step>

  <Step title="Get temporary access token">
    Copy the **Temporary Access Token** (valid for 24 hours):
    
    ```bash
    EAAxxxxxxxxxx...
    ```
    
    <Warning>
      Temporary tokens expire in 24 hours. For production, generate a permanent System User token (Step 4).
    </Warning>
  </Step>
</Steps>

## Step 3: Configure Webhook

<Steps>
  <Step title="Set webhook URL">
    In WhatsApp → Configuration:
    
    - **Callback URL**: `https://yourdomain.com/webhook`
    - **Verify Token**: Create a random string (e.g., `my-secure-verify-token-123`)
  </Step>

  <Step title="Add verify token to environment">
    Update your `.env` file:
    
    ```bash .env
    WHATSAPP_VERIFY_TOKEN=my-secure-verify-token-123
    ```
  </Step>

  <Step title="Verify webhook">
    Click **Verify and Save**. Meta will send a GET request to verify your endpoint.
    
    The webhook endpoint in BookLine handles verification:
    
    ```javascript src/routes/webhook.js:13-25
    router.get('/', (req, res) => {
        const mode = req.query['hub.mode'];
        const token = req.query['hub.verify_token'];
        const challenge = req.query['hub.challenge'];

        if (mode === 'subscribe' && token === env.WHATSAPP_VERIFY_TOKEN) {
            console.log('[Webhook] ✅ Verification successful');
            return res.status(200).send(challenge);
        }

        console.warn('[Webhook] ❌ Verification failed');
        return res.sendStatus(403);
    });
    ```
  </Step>

  <Step title="Subscribe to webhook fields">
    Enable the **messages** field to receive incoming messages.
  </Step>
</Steps>

## Step 4: Generate Permanent Access Token

For production, create a System User token that doesn't expire:

<Steps>
  <Step title="Create System User">
    In Meta Business Settings:
    - Go to **Users → System Users**
    - Click **Add**
    - Name: `BookLine API`
    - Role: **Admin**
  </Step>

  <Step title="Assign app to System User">
    - Click on the System User
    - Click **Add Assets**
    - Select **Apps** → Choose your WhatsApp app
    - Enable **Manage App** permission
  </Step>

  <Step title="Generate access token">
    - Click **Generate New Token**
    - Select your WhatsApp app
    - Permissions: `whatsapp_business_messaging`, `whatsapp_business_management`
    - Token duration: **Never expire** (60 days or custom)
    - Copy and securely store the token
  </Step>

  <Step title="Store in database (per-clinic)">
    BookLine stores access tokens per clinic in the `clinics` table:
    
    ```sql
    UPDATE clinics 
    SET 
      access_token = 'EAAxxxxxxxxxx...',
      phone_number_id = '123456789012345'
    WHERE id = 'clinic-id';
    ```
    
    <Note>
      Each clinic can have its own WhatsApp Business Account with separate credentials.
    </Note>
  </Step>
</Steps>

## Step 5: Configure API Version

BookLine uses WhatsApp Cloud API version 21.0 by default:

```bash .env
WHATSAPP_API_VERSION=v21.0
```

The API base URL is constructed in the WhatsApp client:

```javascript src/whatsapp/client.js:8
const BASE_URL = `https://graph.facebook.com/${env.WHATSAPP_API_VERSION}`;
```

<Note>
  WhatsApp API versions are supported for at least 2 years. Check [Meta's versioning docs](https://developers.facebook.com/docs/graph-api/guides/versioning) for the latest version.
</Note>

## WhatsApp Client Implementation

BookLine's WhatsApp client sends various message types:

### Send Text Message

```javascript src/whatsapp/client.js:14-34
async sendText(phoneNumberId, accessToken, to, text) {
    try {
        await axios.post(
            `${BASE_URL}/${phoneNumberId}/messages`,
            {
                messaging_product: 'whatsapp',
                to,
                type: 'text',
                text: { body: text },
            },
            {
                headers: {
                    Authorization: `Bearer ${accessToken}`,
                    'Content-Type': 'application/json',
                },
            }
        );
    } catch (err) {
        console.error('[WhatsApp] sendText error:', err.response?.data || err.message);
    }
}
```

### Send Interactive List

Used for doctor selection and admin menus:

```javascript src/whatsapp/client.js:39-67
async sendList(phoneNumberId, accessToken, to, { headerText, bodyText, buttonText, sections }) {
    try {
        await axios.post(
            `${BASE_URL}/${phoneNumberId}/messages`,
            {
                messaging_product: 'whatsapp',
                to,
                type: 'interactive',
                interactive: {
                    type: 'list',
                    header: headerText ? { type: 'text', text: headerText } : undefined,
                    body: { text: bodyText },
                    action: {
                        button: buttonText,
                        sections,
                    },
                },
            },
            {
                headers: {
                    Authorization: `Bearer ${accessToken}`,
                    'Content-Type': 'application/json',
                },
            }
        );
    } catch (err) {
        console.error('[WhatsApp] sendList error:', err.response?.data || err.message);
    }
}
```

### Send Interactive Buttons

Used for confirmation flows:

```javascript src/whatsapp/client.js:72-101
async sendButtons(phoneNumberId, accessToken, to, { bodyText, buttons }) {
    try {
        await axios.post(
            `${BASE_URL}/${phoneNumberId}/messages`,
            {
                messaging_product: 'whatsapp',
                to,
                type: 'interactive',
                interactive: {
                    type: 'button',
                    body: { text: bodyText },
                    action: {
                        buttons: buttons.map((btn) => ({
                            type: 'reply',
                            reply: { id: btn.id, title: btn.title },
                        })),
                    },
                },
            },
            {
                headers: {
                    Authorization: `Bearer ${accessToken}`,
                    'Content-Type': 'application/json',
                },
            }
        );
    } catch (err) {
        console.error('[WhatsApp] sendButtons error:', err.response?.data || err.message);
    }
}
```

## Common Issues

<AccordionGroup>
  <Accordion title="Error: (#100) Invalid parameter">
    **Cause**: Phone number format is incorrect
    
    **Solution**: Use international format without `+` or spaces:
    - ✅ Correct: `919876543210`
    - ❌ Wrong: `+91 98765 43210`
  </Accordion>

  <Accordion title="Error: (#131009) Parameter value is not valid">
    **Cause**: Message template or interactive message structure is malformed
    
    **Solution**: Validate JSON structure against [WhatsApp API docs](https://developers.facebook.com/docs/whatsapp/cloud-api/guides/send-messages)
  </Accordion>

  <Accordion title="Messages not received by webhook">
    **Cause**: Webhook not subscribed to `messages` field
    
    **Solution**: 
    - Go to WhatsApp → Configuration → Webhook Fields
    - Ensure **messages** is checked
    - Re-save webhook configuration
  </Accordion>

  <Accordion title="Access token expired">
    **Cause**: Using temporary token (24-hour expiry)
    
    **Solution**: Generate permanent System User token (Step 4)
  </Accordion>

  <Accordion title="Rate limit errors">
    **Cause**: Exceeded messaging limits (1,000/day for test numbers)
    
    **Solution**: 
    - Upgrade to production number
    - Complete Meta Business Verification
    - Request higher limits in WhatsApp dashboard
  </Accordion>
</AccordionGroup>
