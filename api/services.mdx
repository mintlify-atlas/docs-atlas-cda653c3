The service layer contains all the core business logic for the BookLine platform. Each service is a singleton class that handles a specific domain.

## Booking Service

**Location:** `src/services/bookingService.js`

Handles booking creation only after confirmed payment with atomic position assignment.

### processPayment

Process successful payment, assign position atomically, and create booking.

<ParamField path="order" type="object" required>
  The order object containing:
  - `doctor_id` (UUID): Doctor identifier
  - `clinic_id` (UUID): Clinic identifier
  - `date` (string): Booking date in YYYY-MM-DD format
  - `patient_phone` (string): Patient phone number
  - `amount` (integer): Fee in paise
  - `id` (UUID): Order ID
</ParamField>

<ResponseField name="success" type="boolean">
  Whether the booking was created successfully
</ResponseField>

<ResponseField name="booking" type="object">
  The created booking object (if successful)
</ResponseField>

<ResponseField name="position" type="integer">
  The assigned queue position
</ResponseField>

<ResponseField name="reason" type="string">
  Error reason if `success` is false. Possible values:
  - `DOCTOR_CONFIG_NOT_FOUND`
  - `INCREMENT_FAILED`
  - `CAP_EXCEEDED`
  - `BOOKING_CREATION_FAILED`
</ResponseField>

<ResponseField name="refundRequired" type="boolean">
  Whether a refund is needed (cap exceeded)
</ResponseField>

<CodeGroup>
```javascript Usage Example
const bookingService = require('./services/bookingService');

// Process payment after Razorpay webhook
const result = await bookingService.processPayment(order);

if (result.success) {
  console.log(`Booking created! Position: ${result.position}`);
  // Send confirmation to patient
} else if (result.reason === 'CAP_EXCEEDED' && result.refundRequired) {
  // Initiate refund
} else {
  console.error(`Booking failed: ${result.reason}`);
}
```

```javascript Response
{
  success: true,
  booking: {
    id: 'uuid',
    clinic_id: 'uuid',
    doctor_id: 'uuid',
    date: '2026-03-04',
    position: 15,
    patient_phone: '919876543210',
    fee_at_booking_time: 50000,
    platform_fee: 2500,
    clinic_share: 47500,
    order_id: 'uuid'
  },
  position: 15
}
```
</CodeGroup>

## Order Service

**Location:** `src/services/orderService.js`

Handles order creation, lookup, status transitions, and expiry.

### createOrder

Create a Razorpay Payment Link and persist the Order record.

<ParamField path="params" type="object" required>
  Object containing:
  - `clinicId` (UUID): Clinic identifier
  - `doctorId` (UUID): Doctor identifier
  - `patientPhone` (string): Patient phone number
  - `amount` (integer): Amount in paise
  - `dateStr` (string): Booking date (YYYY-MM-DD)
</ParamField>

<ResponseField name="order" type="object">
  The created order record from database
</ResponseField>

<ResponseField name="paymentLink" type="object">
  Razorpay payment link object with `short_url`, `id`, etc.
</ResponseField>

<CodeGroup>
```javascript Usage
const result = await orderService.createOrder({
  clinicId: 'uuid',
  doctorId: 'uuid',
  patientPhone: '919876543210',
  amount: 50000, // ₹500 in paise
  dateStr: '2026-03-04'
});

const { order, paymentLink } = result;
console.log(`Payment link: ${paymentLink.short_url}`);
```
</CodeGroup>

### findByGatewayOrderId

Find an order by its Razorpay payment link ID.

<ParamField path="gatewayOrderId" type="string" required>
  Razorpay payment link ID (e.g., `plink_xxx`)
</ParamField>

### findById

Find an order by its internal UUID.

<ParamField path="orderId" type="UUID" required>
  Internal order ID
</ParamField>

### updateStatus

Update order status with optimistic concurrency control.

<ParamField path="orderId" type="UUID" required>
  Order ID
</ParamField>

<ParamField path="fromStatus" type="string" required>
  Current expected status (for optimistic locking)
</ParamField>

<ParamField path="toStatus" type="string" required>
  Target status. One of: `pending`, `paid`, `expired`, `refunded`
</ParamField>

<ResponseField name="return" type="object | null">
  Updated order object, or null if status didn't match
</ResponseField>

### findStaleOrders

Find pending orders that have passed their expiry time (for notifications).

<ResponseField name="return" type="array">
  Array of expired pending orders
</ResponseField>

### expireStaleOrders

Expire stale pending orders using the database RPC function.

<ResponseField name="return" type="integer">
  Number of orders expired
</ResponseField>

<CodeGroup>
```javascript Cron Job Example
// Run every 5 minutes
const count = await orderService.expireStaleOrders();
if (count > 0) {
  console.log(`Expired ${count} stale orders`);
}
```
</CodeGroup>

### createUpiQr

Create a Razorpay UPI QR Code for single-use payment.

<ParamField path="params" type="object" required>
  - `amount` (integer): Amount in paise
  - `description` (string): QR code description
  - `closeByUnix` (integer): Unix timestamp when QR expires
  - `notes` (object): Metadata to attach to payment
</ParamField>

<ResponseField name="return" type="object | null">
  Object with `id`, `image_url`, or null on failure
</ResponseField>

## Search Service

**Location:** `src/services/searchService.js`

Fuzzy search across clinics and doctors using Fuse.js. Uses a cached in-memory index that refreshes every 5 minutes.

### search

Search for clinics and doctors by free-text query.

<ParamField path="query" type="string" required>
  Search query (minimum 2 characters)
</ParamField>

<ResponseField name="return" type="object">
  Object with:
  - `clinics` (array): Top 5 clinic matches
  - `doctors` (array): Top 5 doctor matches
</ResponseField>

<CodeGroup>
```javascript Usage
const results = await searchService.search('cardio');

console.log(results);
// {
//   clinics: [
//     { id: 'uuid', name: 'Cardio Care Clinic', score: 0.1 }
//   ],
//   doctors: [
//     {
//       id: 'uuid',
//       name: 'Dr. Sharma',
//       specialization: 'Cardiologist',
//       clinicId: 'uuid',
//       clinicName: 'City Hospital',
//       score: 0.05
//     }
//   ]
// }
```
</CodeGroup>

### refreshIndex

Build or rebuild the Fuse search indexes from the database.

<Info>
This is called automatically by `search()` if the cache is stale (> 5 minutes old).
</Info>

### invalidateCache

Force a cache refresh (call after adding/removing clinics or doctors).

## Settlement Service

**Location:** `src/services/settlementService.js`

Tracks platform fees, clinic earnings, and settlement payouts.

**Business Model:**
- Patient pays ₹100 → BookLine Razorpay account
- Platform fee: 5% = ₹5 (BookLine keeps)
- Clinic share: 95% = ₹95 (settled to clinic via UPI/IMPS)
- Razorpay fee: ~2% = ₹2 (deducted from BookLine's balance)
- Net profit: ₹3 per ₹100

### calculateSplit

Calculate platform fee and clinic share for a given amount.

<ParamField path="amountPaise" type="integer" required>
  Total amount in paise
</ParamField>

<ResponseField name="return" type="object">
  Object with:
  - `platformFee` (integer): 5% of amount in paise
  - `clinicShare` (integer): 95% of amount in paise
</ResponseField>

<CodeGroup>
```javascript Example
const split = settlementService.calculateSplit(50000);
console.log(split);
// { platformFee: 2500, clinicShare: 47500 }
// ₹500 → ₹25 platform, ₹475 clinic
```
</CodeGroup>
